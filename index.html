Echo — AI Moderated Chat

Problem

While building a real-time moderation system, I encountered a core ambiguity: what does "offensive" actually mean? Even simple messages like "hi" were sometimes flagged as offensive, while other times they were not. This revealed that LLM outputs are not deterministic and depend heavily on interpretation.

What I experimented with

I started with a simple prompt: classify whether a message is offensive, and if so — generate three alternative respectful versions. However, the results were inconsistent. I experimented with:

  • Different prompt formulations
  • Temperature parameter tuning
  • Direct interaction with the model (Groq) to understand its reasoning
The model explained that short messages like "hi" may be interpreted as impolite due to lack of effort — revealing implicit assumptions.

Solution

I redesigned the prompt into a structured set of explicit rules defining acceptable vs unacceptable messages. This significantly improved consistency. Key insight: LLMs are not binary systems (0/1). They require precise instruction design and constraint definition.

What I learned

Prompt engineering is a core engineering discipline. You can control not only outputs — but reasoning, interpretation, and tone.

FitCheck — AI Virtual Try-On (Pivoted)

Problem

I wanted to understand how clothes (e.g., a trench coat) would look on someone with my proportions — not on tall models.

What I experimented with

Initial approach: 3D try-on using MediaPipe. The challenge was combining a 2D garment with a 3D body — which produced unrealistic results. This exposed two core issues:

  • Dimensional mismatch (2D vs 3D)
  • Lack of structured garment data

Solution (Pivot)

I pivoted to a generative approach:

  • User defines body parameters (height, proportions, skin tone)
  • Garment is provided via URL
  • Image is generated using Gemini API
Instead of simulating physics, I generated realistic approximations tailored to user attributes.

What I learned

Knowing when to pivot is critical. Generative models can outperform simulation when data is incomplete.

Focus Study Platform

Problem

During wartime, students lose focus due to sudden interruptions (e.g., alarms). Returning to concentration is cognitively difficult.

What I built / experimented with

I built a system combining:

  • Task planning
  • Focus timers
  • LLM interaction during interruptions
When an alarm occurs, the user triggers an interaction with Gemini.

Solution

The LLM:

  • Responds empathetically
  • Waits for the user
  • Helps reconstruct what the user was doing before interruption
I explicitly engineered prompts to control tone (empathetic, calming) and guide cognitive recovery. Design decision: manual "alarm" button instead of automatic detection — to give users a sense of control in an uncontrollable situation.

What I learned

LLMs can shape emotional experience, not just outputs. Prompting affects tone, trust, and user recovery.
