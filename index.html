Project 01
AI Virtual Try-On
How does that trench coat actually look on someone my height? I built a tool to answer that — and learned exactly why virtual try-on is a genuinely hard problem.
Image generation Prompt engineering React Parameter tuning Diffusion models
The problem

Online shops show clothes on tall models. A long trench coat that looks elegant on a 180cm model could look completely different on a shorter person — but there's no way to know before buying. I wanted to fix that.

What I tried

First attempt: overlay a 2D garment image onto a 3D body model. The dimension mismatch made it look completely wrong — fabric wouldn't warp naturally to the body. So I pivoted: instead of fitting a garment onto a body, I generated images of a person with specific measurements wearing an item described by its real dimensions — using length, width, and fabric type as input parameters. This produced much more realistic proportional results.

What I learned

Virtual try-on is a genuinely hard problem — body estimation, garment physics, and proportional rendering are active research areas. I got hands-on experience with where current diffusion models succeed and fail, and identified paths forward using ControlNet and garment-specific fine-tuning.

Project 02
Student Focus Platform
A web app to help students stay concentrated and manage stress — built during wartime, tested with real users, iterated based on their feedback.
React Claude Code HTML/CSS Usability testing
The problem

During the war in Israel, students were struggling to focus. Constant news, anxiety, and interrupted routines made studying feel impossible. I wanted to build something that actually helped — not just a productivity timer, but something that acknowledged the emotional context.

How I built it

Built in React with Claude as my core AI collaborator — not just autocomplete, but active collaborators in scaffolding components and debugging logic. Ran qualitative usability sessions with real students, gathered direct feedback, and iterated on UX flows and accessibility across three major design versions.

Impact

Multiple students used the app during exam periods. Their feedback directly shaped the onboarding flow, the stress-check interface, and the focus session timer — each iteration measurably improving retention and ease of use.

Project 03
Echo — AI-moderated chat
A platform for discussing controversial topics, with a real-time AI moderation layer that keeps conversations civil without silencing disagreement.
JavaScript Groq API PostgreSQL Render Real-time moderation
The problem

Online discussions about controversial topics tend to collapse into toxicity fast. Existing platforms either over-censor or do nothing. I wanted to explore whether AI could moderate in a way that felt fair — filtering genuine harm without silencing legitimate disagreement.

How I built it

Integrated the Groq API to run real-time content moderation on every message before it's posted. Built the full stack — frontend, backend, PostgreSQL database, deployed on Render. Collaborated with students across design, social science, and engineering to define what "harmful" actually means in context.

What I learned

Designing AI moderation is as much a product and ethics problem as a technical one. The hardest part wasn't the API integration — it was deciding what the model should and shouldn't flag, and building a system where that judgment felt legitimate to users.
